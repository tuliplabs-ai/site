Tulip

A Python SDK for agent teams that show their work. One Agent class, eight coordination shapes, and a typed event stream behind every run — so the systems you ship are auditable by construction.

Read the docs → GitHub

pip install "tulip-agents[anthropic]"

Why it exists

Most agent frameworks optimize for the demo: a clever answer, fast. That's the easy 80%. The hard part — the part that decides whether you can put an agent anywhere it matters — is everything around the answer. Can you see why it did that? Replay the run? Prove a payment fired exactly once? Pause for a human and resume next week?

Tulip is built for that hard part. The same Agent class scales from a five-line script to an eight-shape multi-agent system, and the whole time it is narrating itself: every reasoning step, tool call, hand-off, and token is a typed event on one stream you can subscribe to, store, and test against.

What's inside
Coordination

Eight shapes, one class

Single agent, sequential and parallel pipelines, loops, orchestrator + specialists, swarm, hand-off, and cross-process A2A — composed from the task, not hand-wired.

Observability

Typed event stream

An opt-in event bus surfaces 40+ canonical events from every layer through one context — no external broker — with an OpenTelemetry hook when you want it.

Reasoning

Reflexion & grounding

Self-evaluation each turn and claims checked against the tool output that produced them, with a typed grounding layer for the strict cases.

Safety

Idempotent tools

Mark a tool idempotent and the loop keys on (name, args) so bookings, payments, and pages never fire twice — even across retries.

Durability

Pause & resume

Checkpoint to Redis, Postgres, S3, and more; interrupt for human approval and resume the run later, exactly where it stopped.

Reach

Provider-agnostic

OpenAI, Anthropic, or any OpenAI-compatible endpoint via base_url. RAG, MCP client + server, and a FastAPI agent server included.

Try it in the browser

The Workbench

A self-contained playground for every Tulip pattern — vanilla-TS UI, a thin Node proxy, and the SDK runner behind it. Bring your own provider key and watch a multi-agent run emit its event stream live. Run it on localhost in three terminals, or as a single Docker container.

Workbench guide → 
# five lines to a working agent
from tulip.agent import Agent

agent = Agent(model="openai:gpt-4o")
print(agent.run_sync("Capital of France?").text)
# → Paris
