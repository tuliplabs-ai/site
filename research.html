Security research

We study the systems
AI actually runs on.

Models get the scrutiny. The infrastructure underneath them — shared GPUs, inference engines, agent runtimes — often doesn't. That's where we look: for the side channels, the trust assumptions, and the failure modes no one has named yet. Then we hand defenders a tool, not just a warning.

Featured · GPU side channels

ingenio

Identify which LLM another tenant is running on a shared GPU. A 1.2-second probe. 94% accuracy. No special privileges. No exploit.

ingenio is a cross-process GPU timing side channel that fingerprints both the inference engine and the language model running on a co-located NVIDIA GPU — from an ordinary unprivileged process sharing the same silicon. The repository is the complete account: fifteen numbered experiments from residual-memory baselines to the spectral, per-token cadence leak that drives the classifier.

True to how we work, the single most useful artifact isn't the attack — it's a 140-line, standard-library audit tool a defender runs on their own infrastructure to measure whether their workload is identifiable from a co-tenant. Alongside it: a mitigation we measured honestly at roughly a quarter of throughput.

Scope, stated up front: this targets shared-GPU configurations where multiple unprivileged processes share one physical GPU (Docker --gpus all, MPS, some shared-tenant cloud offerings, WSL2). It does not defeat dedicated single-tenant GPUs or MIG-partitioned A100/H100s, where compute and memory bandwidth are hardware-isolated.
