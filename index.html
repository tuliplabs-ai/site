What we believe

As AI systems take more consequential actions, the question stops being can they? and becomes can you check? We build for the second question — agents whose every decision is a typed, replayable event, and security work that ships the audit tool, not just the headline.

What we do
Agentic AI

Agents that show their work

Most agent frameworks give you an answer and ask you to trust it. We build the opposite: orchestration where every reasoning step, tool call, and hand-off is a typed event you can stream, replay, and test against — so a multi-agent run is auditable instead of opaque.

Security research

Defender-first, evidence-first

We probe the infrastructure modern AI actually runs on — shared GPUs, inference engines, agent runtimes — and report what we find with measured claims and reproducible experiments. Every finding ships with a tool a defender can run on their own machine.

Tulip

A Python SDK for agent teams that show their work.

One Agent class, eight coordination shapes — single agent, pipeline, debate, approval gate, hand-off, and more — each compiled from the task instead of hand-wired. Every run narrates itself as a typed event stream you can replay, audit, and debug. Provider-agnostic: OpenAI, Anthropic, or bring your own.

from tulip.agent import Agent
from tulip.observability import run_context, get_event_bus

agent = Agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[get_metric, page_oncall],
    grounding=True,   # claims checked vs. tool output
)

async with run_context() as rid:
    await agent.run("p99 spiked to 4.2s — investigate.")
    async for ev in get_event_bus().subscribe(rid):
        print(ev.event_type, ev.data)  # every step, typed

How we work

01Open by default

The work that matters is the work others can build on. Our SDK is Apache-2.0; our research is published in full, tools included.

02Measured claims

Numbers with methods behind them. We state scope and limits as plainly as results — what the finding does not show matters as much as what it does.

03Defender-first

Every security finding ships with something a defender can run today. We study attacks to build the audit, not the exploit.

04Reproducible

Experiments are numbered, scripted, and rerunnable. If you can't reproduce it, we haven't finished.
